# Multi-Departmental-L3-Architecture-DHCP-
Project Buqi: Multi-Departmental Enterprise Core Infrastructure
This repository showcases Phase 1 of Project Apex, a production-ready, hierarchical enterprise campus network designed for maximum address efficiency and rapid data transit. Built within Cisco Packet Tracer, the architecture handles corporate traffic by combining a custom-engineered Variable Length Subnet Masking (VLSM) scheme with high-speed Layer 3 routing operations.

Engineered Topology & LogicThe network discards traditional, bottleneck-prone "Router-on-a-Stick" mechanics. Instead, it utilizes a Multi-Layer Switch (MLS) at its core to perform line-rate Inter-VLAN Routing via Switch Virtual Interfaces (SVIs). To optimize the $192.168.10.0/24$ address space, a precision VLSM layout minimizes IP wastage to less than 15% across four critical corporate domains:  Engineering (VLAN 10): Structured via a $/25$ subnet to support a 100-host requirement with 126 usable slots.  Marketing (VLAN 20): Tailored with a $/26$ subnet for a 50-host department.  Sales (VLAN 30): Allocated a tight $/27$ boundary providing 30 usable addresses.  Admin/HR (VLAN 40): Optimized down to a granular $/28$ subnet for small-team isolation.  Dynamic Enterprise ServicesA key highlight of Project Apex is its decentralized management design. While the Core MLS acts as the default gateway for local endpoints, an autonomous Edge Router handles global addressing. By leveraging Cisco IP Helper-Addresses on each SVI, DHCP broadcast requests are securely targeted across a $/30$ point-to-point WAN backbone link ($192.168.10.240/30$). The Edge Router then dynamically leases precise IP configurations back to the endpoints based on their originating VLAN membership. Furthermore, explicit static routes are programmed at the edge to guarantee symmetrical packet return paths through the MLS core.  Scalability BlueprintThis infrastructure forms a robust foundation for modern enterprise scaling. The layout intentionally separates the internal routing fabric from external edge constraints, establishing a direct roadmap for Phase 2 deployments, which will introduce Inside Global NAT/PAT translation matrices, dedicated DMZ server farmlands, and adaptive Cisco ASA Firewall security zones.
